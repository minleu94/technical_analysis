# Architecture Governance Lifecycle

## 1) Audit（掃描）
- 定期盤點模組責任與依賴關係  
- 識別跨層依賴與 Legacy 擴散  

## 2) Policy（定義）
- 建立 Architecture Boundary Policy  
- 明確 Allowed / Forbidden / Exceptions  

## 3) Checklist（落地）
- 將政策轉為可勾選的治理清單  
- 插入到新功能前/PR Review 前  

## 4) Evolution（Phase C Refactor）
- 在治理規範穩定後，進入結構收斂  
- 執行 Legacy 移除與層級修正
