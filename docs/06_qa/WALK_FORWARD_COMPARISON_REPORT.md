# Fixed vs Quantile 驗證報告

## 1. 驗證狀態

本文件記錄 Strategy & Scoring Governance 的機制與回歸驗證。

目前已完成：

- fixed 模式訊號相容性測試。
- quantile Expanding T-1、60 個有效觀測暖機與未來資料不變性測試。
- walk-forward 測試窗承接該 fold 訓練歷史、但只計算 OOS 撮合與績效的時間軸回歸測試。
- 推薦 eligible universe empirical CDF、同分同百分位、穩定排序與最小母體防禦測試。
- DTO / Preset / StrategyVersion round-trip。
- UI、mypy、金融 float boundary 與資料更新工作台 QA。
- 10 檔股票、單一策略與固定成本假設的可追溯 OOS pilot，包含資料 SHA-256、OOS 報酬、最大回撤、Sharpe、交易次數與 Regime 分層。
- 最低交易樣本與 Regime coverage Gate 均已通過。

本次實證限制：

- 股票池仍限 10 檔、單一策略與單一成本組合，不代表跨策略或全市場統計顯著性。
- 尚未完成 benchmark-relative attribution 與跨 run 比較。

實證結果未顯示 quantile 的平均 OOS Sharpe 優於 fixed，因此不能宣稱 quantile 已改善績效或穩健度，也不能作為把 quantile 設為預設模式的依據。

## 2. 已驗證契約

### 2.1 回測時間序列

- 未提供 `threshold_mode` 時等同 `fixed`。
- fixed 使用原始分數與 `buy_score` / `sell_score` 比較，不以量化結果改變訊號。
- quantile 的 T 日門檻只使用 T-1 以前的有效 `score_bp`。
- 暖機期固定為 60 個有效觀測值；第 61 個有效觀測才可判定。
- `buy_quantile_bp` 必須大於 `sell_quantile_bp`。
- `quantile_method` 第一版只接受 `nearest_rank`。
- 在序列尾端附加未來資料，不得改變既有日期的門檻與訊號。

### 2.2 推薦橫斷面

- 缺少 `recommendation_ranking` 時維持 fixed 排序。
- quantile 必須明確提供最低百分位、最小母體與排名方法。
- empirical CDF 使用 `bisect_right`，同分股票取得相同百分位。
- 百分位母體在套用 `top_n` 前建立。
- 母體不足時拋出 `RecommendationUniverseTooSmallError`，不降級為 fixed。
- 最終排序為 `total_score desc, stock_code asc`。

## 4. 正式實證驗收契約

正式 walk-forward 比較至少要固定：

- 股票池與資料版本。
- 訓練、驗證、測試窗口。
- 資料截止日。
- 手續費、滑價與成交價格假設。
- fixed / quantile 除門檻以外完全相同的策略設定。
- 報酬、最大回撤、Sharpe、交易次數、暖機後有效日數、推薦通過率與換手。

2026-06-14 已依上述契約完成本段 pilot：

- 股票池由 5 檔擴大為 10 檔，fixed 門檻固定為 55/45。
- fixed 與 quantile 使用相同股票池、資料版本、窗口、成本與成交假設。
- 最低交易樣本 Gate 與 Regime coverage Gate 均通過。
- 逐日 Regime 只使用決策日當下可得資料；同日大盤缺列時只向前沿用最近標籤。
- 報告保留 quantile 未優於 fixed 的負向結果。

Quantile 維持 opt-in；後續 Cross-run、benchmark-relative attribution 與更大股票池屬下一階段研究，不阻塞本段 Gate 結案。

### 5.5 判讀與限制

- **結論**：Quantile 的平均 OOS Sharpe 未優於 fixed；目前結果未證明 quantile 改善報酬、Sharpe 或穩健度。
- 這是 10 檔股票、單一策略與單一成本組合的 pilot，提供真實 OOS 的 regime 分層統計。
- Daily regime 依日期遞增、使用隔離的歷史狀態計算；每個標籤只讀取該日以前的大盤資料。
- 個股交易日若缺少同日大盤列，沿用該決策日以前最近一筆可用 Regime，不使用未來日期回填。
- Quantile 維持 opt-in；在整體 fixed 有足夠交易樣本且 quantile 未能展現顯著績效提升前，維持原設定。
