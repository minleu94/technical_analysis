## 使用方式

每一項請依實際操作填入：

- 狀態：`待測` / `通過` / `失敗` / `需確認` / `跳過`
- 使用者回覆：記錄實測答案、錯誤訊息、畫面現象或操作結果。
- 證據 / 備註：可填截圖路徑、log 摘要、資料日期、輸出檔名或重現條件。

若遇到失敗，先記錄，不在本檔直接改程式。後續修復任務另開 scope。

### Issue 狀態規則

後續修復時，本檔採用 issue 記錄簿模式：

| 狀態 | 意義 |
|---|---|
| 已記錄 | 已由人工驗證發現並寫入本檔，尚未修正。 |
| 修正中 | 已開始分析或修改，但尚未完成。 |
| 已修正待驗證 | 程式已修正，但尚未由使用者重新操作驗證。 |
| 驗證通過 | 使用者已重新驗證，確認問題消失。 |
| 重開 | 修正後仍重現，或衍生出同一根因問題。 |

## 備註

- 高風險資料操作（強制全量、清空、刪除正式紀錄）優先只驗證「取消 / 防呆 / 提示」流程；若要實際執行，需另行確認並先備份。
